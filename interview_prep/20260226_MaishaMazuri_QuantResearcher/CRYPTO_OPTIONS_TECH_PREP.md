### 问题2: "Black-Scholes implementation - did you account for crypto specific features?"

**可能的追问**:
- 传统Black-Scholes假设连续交易，crypto是24/7的
- Crypto的高波动率怎么处理？
- 你怎么处理IV的计算？

**标准回答**:
```
"我实现的是标准Black-Scholes，但我清楚它的局限性:

1. **Crypto的特殊性**: 
   - 24/7交易意味着无风险利率模型需要调整
   - 高波动率下BS的假设(对数正态分布)可能不够准确
   - 我目前的处理是用更高的波动率输入，但承认这需要更复杂的模型

2. **IV计算**:
   - 我用二分查找实现IV求解器，从市场价格反推IV
   - 也看了Newton-Raphson方法，但收敛性问题还没完全解决
   - 实际中发现IV曲面在crypto中非常陡峭，这是我知道但没有完全解决的问题

坦率地说，我现在的实现是'能work的教学版本'，离生产级还有距离。
比如我没有实现局部波动率或随机波动率模型，这些是我知道需要但还没做的。"
```

**技术细节 (如果被深挖)**:
- 二分查找 IV solver: `IV = find_root(market_price - BS_price(S, K, T, r, sigma), a, b)`
- 收敛条件: `abs(diff) < 1e-6` 或最大迭代次数
- 边界处理: sigma_min=0.001, sigma_max=5.0

### 问题3: "Walk me through your Greeks calculation."

**可能的追问**:
- Delta的计算公式
- Gamma和Vega的关系
- Theta在时间衰减中的表现
- 你怎么处理分档(阶梯)期权的希腊字母？

**标准回答**:
```
"我实现了完整的希腊字母:

- **Delta**: ∂V/∂S - 用解析公式，也做了有限差分验证
- **Gamma**: ∂²V/∂S² - 二阶导，在ATM附近最大
- **Theta**: ∂V/∂t - 时间衰减，到期前加速
- **Vega**: ∂V/∂σ - 对波动率的敏感度
- **Rho**: ∂V/∂r - 利率敏感度，crypto中相对次要

我用的都是闭式解(closed-form)，不是数值方法。

**一个我发现的细节**: 在crypto期权中，由于底层波动率很高，
Gamma风险特别重要。我设置了Gamma限制作为风控的一部分。

**我不知道的**: 分档期权的希腊字母怎么处理？
如果你的期权在特定价格触发不同payoff，
我的理解是此时希腊字母可能不连续，但我没有实现这种处理。"
```

**公式备忘** (如果被要求写出来):
```python
# Delta
Delta_call = N(d1)
Delta_put = N(d1) - 1

# Gamma (same for call and put)
Gamma = N_prime(d1) / (S * sigma * sqrt(T))

# Theta
Theta_call = -S * N_prime(d1) * sigma / (2 * sqrt(T)) - r * K * exp(-r*T) * N(d2)

# Vega (same for call and put)
Vega = S * N_prime(d1) * sqrt(T)

# 其中 N() 是标准正态CDF, N_prime() 是PDF
d1 = (ln(S/K) + (r + sigma^2/2)*T) / (sigma*sqrt(T))
d2 = d1 - sigma*sqrt(T)
```

### 3. "What's the difference between realized vol and implied vol?"
**标准答案**:
```
"RV是过去实际发生的波动率，IV是市场对未来波动率的预期。

在期权交易中:
- 如果RV > IV，卖方亏，买方赚(long gamma)
- 如果RV < IV，卖方赚，买方亏

做市商本质上是在卖出波动率，赚取IV premium，
但如果RV实际更高，就会亏损。

我的系统监控这个spread，但目前没有成熟的交易策略。"
```
