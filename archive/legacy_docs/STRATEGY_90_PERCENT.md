## 🎯 90% vs 100% 的区别

| 环节 | 100%自动化 | 90%自动化 (推荐) | 原因 |
|------|-----------|-----------------|------|
| 职位发现 | ✅ 自动爬取 | ✅ 自动爬取 | 无风险，可完全自动 |
| 硬性筛选 | ✅ 规则过滤 | ✅ 规则过滤 | 确定性规则，零错误 |
| AI评分 | ✅ 自动评分 | ✅ 自动评分 | 模型可靠，可批量处理 |
| 简历生成 | ✅ 自动定制 | ✅ 自动定制 | 模板+AI，质量可控 |
| **简历审核** | ❌ 自动投递 | ⚠️ **人工确认** | 避免低级错误，保护品牌 |
| **投递执行** | ❌ 全自动 | ⚠️ **半自动辅助** | 应对复杂表单，处理异常 |
| 状态追踪 | ✅ 自动记录 | ✅ 自动记录 | 无风险，可完全自动 |

## 📊 每日工作流程 (实际操作)

### 早上 (自动完成)
```bash
# 定时任务运行 (你无需操作)
python job_hunter_cli.py daily
```
- 爬取新职位 (LinkedIn, IamExpat)
- 硬性筛选 (排除 Dutch required, 10+ years等)
- AI评分 (0-10分)
- 为高匹配职位生成定制简历

### 晚上 (你只需5-10分钟)
```bash
# 查看今日推荐
python job_hunter_cli.py review
```
输出示例:

按 `Y` 自动打开申请页面，预填信息，你只需:
1. 快速检查表单
2. 点击提交

## 🛠️ 系统架构

### 核心组件

```
job-hunter/
├── 📁 scripts/
│   ├── crawler.py           # 职位爬取 (全自动)
│   ├── hard_filter.py       # 硬性筛选 (全自动)
│   ├── ai_analyzer.py       # AI评分 (全自动)
│   ├── resume_generator.py  # 简历生成 (全自动)
│   └── apply_assistant.py   # 投递辅助 (半自动)
│
├── 📁 data/
│   ├── jobs_pending.json    # 待审核职位
│   ├── jobs_applied.json    # 已投递记录
│   └── stats.json           # 统计数据
│
└── 📁 output/
    └── Fei_Huang_*.pdf      # 生成的简历
```

### 关键设计决策

| 决策 | 方案 | 理由 |
|------|------|------|
| 爬取频率 | 每日1次 | 平衡新鲜度和API限制 |
| 评分阈值 | ≥6.0生成简历 | 保证质量，减少噪音 |
| 人工确认 | 强制确认 | 避免错误投递，保护专业形象 |
| 投递辅助 | 浏览器自动化 | 预填信息，但不自动提交 |
| 错误处理 | 记录+跳过 | 不中断流程，人工后续处理 |

## ⚠️ 风险控制

### 为什么不做100%自动化?

1. **简历错误风险**
   - AI生成的简历可能有事实错误
   - 自动投递前不检查 = 专业形象受损
   - 5秒的人工扫一眼可以避免尴尬

2. **申请页面多样性**
   - 每个公司的申请流程都不同
   - Workday, Greenhouse, Lever 各有 quirks
   - 全自动容易卡在奇怪的地方

3. **验证码和反爬**
   - 频繁自动操作可能触发验证
   - 人工介入可以处理异常情况

4. **策略调整灵活性**
   - 市场变化时需要快速调整策略
   - 人工确认层提供这个灵活性

## 💡 关键成功因素

1. **简历质量 > 投递数量**
   - 宁可少投，也要保证每份简历都精准
   - 人工确认层确保这一点

2. **持续优化筛选规则**
   - 根据反馈调整硬性筛选
   - 减少误杀，提高效率

3. **快速迭代**
   - 先跑起来，再优化
   - 每周回顾数据，调整策略
