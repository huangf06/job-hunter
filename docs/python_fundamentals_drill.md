# Python Fundamentals Drill — Daily Practice (2026-03-05)

## Purpose
Build muscle memory for basic Python coding under interview pressure.

**Target**: Write bug-free implementations of simple algorithms in 5 minutes, no AI, no autocomplete.

**Trigger**: Two consecutive live coding failures on trivial problems (FareHarbor 2026-02-23, 2026-03-05).

---

## Rules
1. **30 minutes daily, hand-write or type without IDE autocomplete**
2. **Zero AI assistance** — no Claude, no Copilot, no Stack Overflow
3. **Time each problem** — track improvement over days
4. **Test your code manually** — trace through with example inputs
5. **If stuck > 10 min, mark as "review needed" and move on**

## Week 1: Core Algorithms (Easy)

### Day 1: Distance & Sorting
1. **Euclidean distance** between two points (2D, then 3D)
2. **Manhattan distance** between two points
3. **Sort list of tuples** by second element
4. **Sort list of dicts** by specific key

### Day 2: K-Nearest Neighbors
1. **KNN from scratch** (given X_train, y_train, x_test, k)
   - Calculate distances
   - Sort by distance
   - Take top k
   - Return majority vote (classification) or mean (regression)
2. **Weighted KNN** (distance-weighted voting)

### Day 3: Array Manipulation
1. **Two sum** (return indices of two numbers that add to target)
2. **Group by key** (list of dicts → dict of lists)
3. **Find duplicates** in array
4. **Rotate array** by k positions

### Day 4: String & List Operations
1. **Reverse string** (in-place if possible)
2. **Check palindrome**
3. **Merge two sorted lists**
4. **Remove duplicates** from sorted array (in-place)

### Day 5: Binary Search
1. **Binary search** (iterative)
2. **Binary search** (recursive)
3. **Find first occurrence** of target in sorted array
4. **Find insertion position** for target

### Day 6: Two Pointers
1. **Valid palindrome** (ignore non-alphanumeric)
2. **Container with most water**
3. **Remove element** from array (in-place)
4. **Move zeros** to end of array

### Day 7: Review & Speed Test

- Time yourself on each
- Goal: < 5 min per problem

## Interview Warm-Up Routine (Day of Interview)

**30 minutes before interview:**
1. Solve 2-3 easy problems from above (ones you've done before)
2. Focus on **writing clean code out loud** — practice thinking aloud
3. Test your code manually with edge cases
4. Goal: Get into "flow state" before the real interview

---

## Red Flags (When to Escalate)

If after 2 weeks you still:
- Take > 10 min on Day 1-7 problems
- Cannot implement KNN in < 5 min
- Panic when asked to code without IDE

→ Consider hiring a coding interview coach or doing more intensive practice (e.g., daily mock interviews).
