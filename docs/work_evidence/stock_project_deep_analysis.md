### 2.2 核心算法与逻辑

#### A. 涨停板筛选器 (ZTB.py)

**业务逻辑：**
```python
# 1. 获取当日所有股票数据
df = pro.daily(trade_date=today)

# 2. 筛选涨幅 > 9.9% 的股票（涨停阈值）
top_high = df[df.changepercent > 9.9]

# 3. 识别"一字涨停"（最强动量信号）
top_high['one'] = top_high['low'] == top_high['high']

# 4. 排除停牌股票（换手率 = 0）
top_high = top_high[top_high['turnoverratio'] > 0]

# 5. 按换手率排序，导出Excel
top_high.sort_values('turnoverratio').to_excel('ztb.xlsx')
```

**技术亮点：**
- ✅ 理解中国股市涨停板机制（±10%限制）
- ✅ 识别"一字涨停"（开盘=最高=最低=收盘）
- ✅ 自动排除停牌股票

**局限性：**
- ❌ 无历史回测 - 不知道涨停板次日表现
- ❌ 无风险控制 - 没有止损、仓位管理
- ❌ 无信号生成 - 只是筛选，没有买卖决策

#### B. 龙虎榜解析器 (AbstractLHB.py)

**技术复杂度：最高**

**挑战：**
1. **多数据源** - 沪深交易所、创业板格式不同
2. **非结构化数据** - 纯文本报告，无JSON/XML
3. **动态格式** - 交易所可能随时调整格式

**解决方案：状态机解析器**

```python
class AbstractLHB:
    def __init__(self):
        self.state = 'readStocks'  # 初始状态
        self.stocks = []
        self.current_stock = {}

    def parse_line(self, line):
        if self.state == 'readStocks':
            # 解析股票代码和名称
            if '买入' in line or '卖出' in line:
                self.state = 'readBS'

        elif self.state == 'readBS':
            # 解析买卖方向
            if '机构专用' in line:
                self.state = 'readBuy'

        elif self.state == 'readBuy':
            # 解析买入金额
            self.current_stock['buy'] = extract_amount(line)
            self.state = 'readSell'

        elif self.state == 'readSell':
            # 解析卖出金额
            self.current_stock['sell'] = extract_amount(line)
            self.stocks.append(self.current_stock)
            self.state = 'readStocks'  # 回到初始状态
```

**技术亮点：**
- ✅ **状态机设计** - 处理复杂的文本解析逻辑
- ✅ **多格式兼容** - 支持3种交易所格式
- ✅ **增量采集** - 记录最后采集日期，避免重复

**数据输出示例：**
```csv
日期,代码,名称,净流入流出,流入,流出,偏离值,成交量,成交金额(万元)
2019-08-05,'600547.SH','山东黄金',+280000000,+280000000,0,+10.01%,12345678,987654
```

**业务价值：**
- 龙虎榜 = 机构席位交易明细
- 机构净买入 → 可能是"聪明钱"信号
- 连续上榜 → 可能有主力资金运作
