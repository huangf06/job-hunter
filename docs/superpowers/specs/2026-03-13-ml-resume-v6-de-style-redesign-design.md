# ML Resume V6 DE-Style Redesign Design

## Goal

Retarget `templates/Fei_Huang_ML_Resume_v6.svg` into a stronger ML / Applied AI / data-plus-ML hybrid resume while keeping the restrained two-column visual language of `templates/Fei_Huang_DE_Resume.svg`.

## Constraints

- Keep all facts, dates, titles, metrics, and scope grounded in the current resume evidence.
- Preserve the reference resume's conservative recruiter-friendly tone and layout family.
- Do not introduce a more aggressive hero layout or non-reference visual system.
- Keep the output to one page in English SVG format.

## Approved Direction

- Use the DE resume as the style reference for spacing, typography, and module order.
- Rebuild the ML story around one line: production decision systems first, then quantitative rigor, then recent AI training.
- Keep the standard structure:
  - Header
  - Bio
  - Experience
  - Education
  - Projects
  - Technical Skills
  - Languages

## Narrative Decisions

1. Header / opening
   - Keep the same restrained header treatment as the DE reference.
   - Let the opening summary carry the ML positioning instead of adding a new hero banner.

2. Experience weighting
   - `GLP Technology` remains the strongest proof point and should read as end-to-end production decision-system ownership.
   - `BQ Investment` should emphasize deployed research, live capital, and systematic validation.
   - `Independent Quantitative Researcher` stays, but in compressed form to signal continuous self-directed research rather than filler.
   - `Ele.me` stays compressed and framed as early decision/rules/fraud-scale experience.

3. Right-column support
   - `VU Amsterdam M.Sc. AI` remains visible but subordinate to commercial experience.
   - Keep only the recent ML projects that best support current capability:
     - uncertainty quantification benchmark
     - Expedia recommendation
   - Skills should function as a hiring-support scan zone, not a keyword dump.

## Copy Strategy

- Use concise, senior, factual phrasing.
- Lead bullets with system scope or business function, then method, then outcome.
- Avoid coursework feel and generic ML buzzwords.
- Keep the language aligned with hiring-manager scan behavior in the first 30 seconds.
